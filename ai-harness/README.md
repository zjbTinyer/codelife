# 🛡️ AI 评估与控制工程

> 让 AI 输出可控、可靠、可量化

## 模块定位

这个模块关注的是**你作为开发者，如何评估和控制 AI/LLM 系统的行为**。它不依赖任何特定框架，是通用的工程方法论。

如果 Agent 开发教你"怎么让 AI 做事"，那这个模块教你"怎么确保 AI 把事做对"。

## 为什么需要这个模块？

```
没有评估和控制:
  改了 Prompt → 不知道效果变好还是变差
  模型升级    → 不知道会不会出问题
  上线部署    → 心里没底，祈祷别出bug

有了评估和控制:
  改了 Prompt → 跑评估套件，通过率 98% ✅
  模型升级    → 红队测试通过，安全评分 0.95 ✅
  上线部署    → 金丝雀观察正常，全量发布 ✅
```

## 🔗 与其他模块的关系

```
ai-harness/ (本模块)
  ├── 教你如何评估和防护 AI 系统
  │
  ├── 与 claude-code-harness/ 的关系：
  │   claude-code-harness 是 Claude Code 如何内部控制工具的安全
  │   本模块是你作为开发者应该如何设计这些控制
  │   一个是"实现"，一个是"方法论"
  │
  └── 与 harness-cicd/ 的关系：
      harness-cicd 负责把 AI 应用部署到生产
      本模块的评估和测试确保部署上去的是高质量的
      部署策略（金丝雀）+ 评估门禁 = 安全上线
```

## 🎯 学完这个模块你会

- ✅ 设计一套完整的 AI 应用评估体系
- ✅ 实现多层安全护栏（输入→对话→工具→输出）
- ✅ 用熔断、重试、降级保证 Agent 服务不崩溃
- ✅ 像管理代码一样管理 Prompt（版本化、A/B测试、CI/CD）
- ✅ 搭建 LLM 应用的可观测性体系（追踪+指标+日志）

## 🛠️ 核心代码示例

模块中的代码使用 **Python**，涉及的库包括：
- `anthropic` / `openai` — LLM API
- `chromadb` — 向量数据库（评估数据集存储）
- `prometheus_client` — 指标暴露

所有示例都是**独立可运行的概念代码**，方便你理解核心机制。
