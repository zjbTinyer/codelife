# ⚙️ Claude Code Harness 运行时

> 深入理解 Claude Code 内部的工具管理、权限控制、Hooks 机制

## 模块定位

这个模块聚焦于 **Claude Code 内部的 Harness 层**——它是如何安全管理工具调用、拦截危险操作、通过 Hooks 扩展功能的。

学习这个模块让你不仅能**用** Claude Code，还能**理解它为什么这样设计**，甚至**借鉴其设计思想到自己的 Agent 系统中**。

## 为什么需要这个模块？

```
只会用工具:
  "Claude 可以执行命令，用就行了"

理解 Harness 层:
  "Claude 执行每个命令前会经过：
   参数校验 → 安全扫描 → 权限检查 → 用户审批 → 沙箱 → 执行 → 审计"
  → 你可以为自己的 Agent 系统设计同样的安全机制
```

## 🛠️ 涉及的技术

- **Hook 脚本**: Python / Shell（通过环境变量与 Harness 通信）
- **配置格式**: JSON (settings.json)
- **安全策略**: 正则匹配、命令前缀白名单、OPA/Rego

## 💡 关键设计思想

```
1. 纵深防御 — 不是一层安全检查，是多层
2. 默认不信任 — 未知操作默认需要审批
3. 用户可控 — 关键决策交给用户，不替用户做决定
4. 可扩展 — Hooks 机制让用户自定义行为
5. 分层配置 — 团队共享 + 个人覆盖，互不干扰
```
